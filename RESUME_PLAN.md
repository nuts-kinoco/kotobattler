## 🔮 1. コトバトラーのアプリ概要
コトバトラーは、ボイスチャット（VC）やオンラインゲーム中に「デスした瞬間」などのわずかな合間に、片手でそっとお題をめくって会話を豊かにする、**超高級・ローカル完結型のVC補助カードアプリケーション**です。
*   **技術スタック**: Next.js (App Router), React, Framer Motion, Tailwind CSS, Lucide React
*   **プライバシー重視**: すべてのデータは外部サーバーへ送信されず、ブラウザの `LocalStorage` のみに安全かつ堅牢に保存されます。

ご主人様との完璧なペアプログラミングによって、以下のすべてのアップグレードと機能が100%完璧に実装され、プロダクションビルドもノーエラーで完了・GitHubへプッシュ済みです。

### 💾 A. 統合JSON完全フルバックアップ ＆ CSVの役割分担
*   **CSVインポート/書き出し**: `DeckManager`（カードロッカー）にマウント。お題カード（テキスト、星、タグ、空気感）の編集・外部やり取り専用にクリーンに分離。
*   **JSON完全フルバックアップ**: コントロールパネルの「設定 & ショートカット」タブ内に一元化。お題カード、デッキ、メンバーメモ、会話履歴、テーマ設定、セッション状況など、**コトバトラーでの全ての体験データを一撃で完全パックしてPCへ保存・復元**します（容量は数KB〜数十KBと非常に軽量なため、ZIPにせず単一のJSONだけで一瞬で動作します）。

### 🔄 B. 「前回のメンバーを記憶する」トグルスイッチ (デフォルトON)
*   設定タブにネオングリーンの美しいトグルスイッチを配置。セッションを終了しても前回のVCメンバーをLocalStorage内に安全に保持し、次回開始時にワンクリックでVCを再開できる、人間に優しい極上のUXを実装。

### ❌ C. メンバー選択時の「ポップアップなし一撃非表示バツボタン」
*   セッション開始前のメンバーチェックリストにおいて、ホバー時にふわっと浮かび上がる `X` ボタンを実装。
*   ロッカーからの物理削除ではなく、**「今回のセッションの選択候補から一時的に消す（非表示にする）」**というスマートな一時非表示リスト（`excludedPersonIds`）制御。確認ポップアップを一切挟まず、サクッと候補を消去・選択解除できます。セッション開始・キャンセル時に自動リセットされデータは完全安全です。

### 📱 D. Apple品質の「タスクキル ＆ スライドアニメーション」の調律
*   **「話した！」 (Used)**: iPhoneのアプリを真上にシュッと投げ捨てるかのような、加速を伴うタスクキルアニメーション（`y: -800`）。Reactの非同期ステートバッチングのズレを `60ms` の遅延制御（`setTimeout`）で完全に手なずけ、100%確実に真上へ飛んでいく職人技の同期を完了。
*   **「パス」 (Skip)**: 右側へすーっと風のように滑らかにスライド・フェードアウト（`x: 500`）。新しいカードは左下から優しく入ってきます。
*   **「切り替え（選択時）」の超静寂化**: カルーセルで単にお題を切り替える際の無駄なフニャフニャした揺れ・ばね演算・遅延（`delay`）を完全に排除。もともとの「スッ…」と静かに滑るような美しい横スライド（CSSクラス制御）のみに調律完了。

### 🧹 E. ゲーミング感の排除と視認性・表記のクリーンアップ
*   グラデーションカラーのタイトルを落ち着いたモノクロームのソリッドカラーに変更し、視覚的ノイズをシャットアウト。
*   ライトモード時の黄色（警告カード）のテキストカラーを深みのあるアンバーブラウンに変更し、白背景でも非常にクッキリ読める極上コントラストへ。

## 🚀 3. 次なる大きな目標：空気感フィルターの動的マスタ化
現在、お題カードに紐づく空気感フィルター（`AirSuitability`）は `'はじめまして' | '連勝中' | '連敗中' | '不穏' | 'お見合い' | '深夜帯' | '普通'` といった固定の文字列のユニオン型ですが、これをお客様（ご主人様）自身がアプリのUIから自由に追加、削除、リネーム、並び替えできる動的な仕組みへと大進化させます。

### 🛠️ 設計方針と実装ロードマップ

#### ① データベース（LocalStorage ＆ 型定義）の拡張
*   空気感（フィルター）の定義リストを、新たに `squid_air_modes`（LocalStorageのキー）として保存できるようにします。
*   初期値として、ご主人様と考え出したゲーム特化型のプリセットをロードさせます：
    ```json
    [
      { "id": "air-intro", "name": "はじめまして" },
      { "id": "air-win", "name": "連勝中" },
      { "id": "air-lose", "name": "連敗中" },
      { "id": "air-clean", "name": "不穏" },
      { "id": "air-chat", "name": "お見合い" },
      { "id": "air-midnight", "name": "深夜帯" },
      { "id": "air-normal", "name": "普通" }
    ]
    ```
*   `Card` インターフェースの `airSuitability` の型を、固定の union型 から、動的に管理される `string`（空気感の `id` もしくは `name`）へと安全に拡張・フォールバックできるように移行します。

#### ② 設定 ＆ ロッカー整理画面への「空気感マネージャー」の新設
*   設定タブ、またはロッカー整理（`DeckManager`）画面の中に、**「空気感フィルターの管理」エリア**を美しく新設。
*   「新規空気感の追加（プラスボタン）」「不要な空気感の削除（ゴミ箱）」「名前のインライン編集」ができるスタイリッシュなUIを構築します。

#### ③ 完全フルバックアップ（JSON）への統合
*   新設する動的な空気感リスト（`airModes`）も、`KotobattlerBackupData` の中に自動的に組み込み、JSONバックアップの完全保存・完全復元の対象に含めることで、PCを引っ越しても自分で作ったフィルターが1秒で完璧に復活するようにします。
